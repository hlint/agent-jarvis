# Context Prune

此处记录如何更好地使用系统工具 `context-prune`（上下文裁剪）。

## 何时执行

- **仅在系统空闲时**：收到系统事件 `data.type="system-inactive"` 时，先更新记忆、写日记，再执行裁剪。**除非用户明确要求**，否则不在对话进行中执行。
- 不在「刚聊几句」的短对话中裁剪；也不在一轮思考或工具链执行到一半时裁剪。

## 保留哪些消息（keepEntryIds）

系统会**强制保留最近 8 条**，无需在 `keepEntryIds` 中指定。你只需传入希望**额外保留**的、更早的条目 id。

建议额外保留：

- 与当前任务强相关的少量历史：用户明确提到的需求、关键决策、错误信息等，按需 1～3 条即可。
- 若存在与长期记忆/日记相关的 system-event，可考虑保留。

不必保留：

- 早期的寒暄、已完成的子任务细节、大量重复的 thinking/tool-call 中间步骤。

传入的 `keepEntryIds` 必须是当前 `dialogHistory` 里真实存在的 entry 的 `id`；不存在的 id 会被忽略。
