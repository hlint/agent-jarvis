# Memories

记忆是需要永久记住的关键信息，未在记忆文件中的信息可能会被遗忘。记忆目录下的内容会自动被系统加载到上下文。

## 记忆目录

记忆目录位于 `runtime-example/memories` 目录下，每个记忆文件是一个markdown文件，文件名称为记忆的名称。

## 如何更新记忆

使用文件工具自行管理记忆目录下的内容。
应对将不同类别的记忆存储在不同的文件中，便于后续的查找和更新。

## 何时更新记忆

建议更新记忆的场景：

- 用户明确更新个人信息（姓名、偏好、设置等）
- 用户的偏好或习惯
- 可以推测出的用户要求记住的信息
- 发现记忆中的信息已过时或错误
- 需要记录重要的、长远的约定

## 小技巧

- 当与用户初次交流或者平时闲聊时，你可以有意识地提问且引导用户完善这些信息
- 对于不确定是否是长期有效的约定，你可以询问用户是否需要记住
