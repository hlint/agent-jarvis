# Diary

此处记录日记的约定与用法：日记以按日分文件的形式存放在 runtime 的 `diaries/YYYY/MM/DD.md`，仅追加不覆盖。系统会将最近 3 天的日记内容自动加载到上下文，用于衔接之前的进展。

## 存储位置与格式

- **路径**：`diaries/YYYY/MM/DD.md`（相对于 runtime），月、日两位补零，例如 `diaries/2026/02/17.md`。
- **单条格式**：每条占一行，格式为 `**Weekday, Month DD, YYYY HH:MM AM/PM:** 内容`（与系统 当前时间格式一致，如 `**Friday, February 17, 2026 21:35 PM:** 内容`）。条与条之间空一行。

示例：

```
**Friday, January 01, 2026 01:00 AM:** Foo

**Friday, January 01, 2026 02:00 AM:** Bar

```

## 如何写日记

使用 [append-to-file]：路径为当日文件 `diaries/YYYY/MM/DD.md`，内容为「`**时间:** 内容`」格式的单行。若当日文件不存在，会随追加自动创建。时间可用当前对话或系统提供的当前时间，格式与上一致。

## 何时写、写什么

- **判断是否值得写**：完成有意义的任务、学到新知识、解决重要问题后写；不记录琐碎、无信息量的内容；聚焦对后续对话有帮助的上下文。
- **何时写**：**仅在系统空闲时**（收到 `system-inactive` 等系统事件）执行写日记和上下文裁剪；先写日记，再裁剪。除非用户明确要求，否则不在对话进行中主动写日记或裁剪。
- **篇幅简短**：每次不必写多，甚至一句话即可，例如：「帮用户修复了 XXX 的 bug，原因是 YYY」「了解了 ZZZ 的 API 用法」。

系统会将**最近 3 天**（含当天）的日记文件内容拼接后注入上下文，便于延续之前的进展。
